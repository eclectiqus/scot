Name

Scot::Model::Alertgroup

Description

The model of an individual alertgroup. Alertgroups are aggregations of alerts. Why? Splunk and other detectors often send in reports with multiple rows of data. Each row is broken into an alert. This helps with various things from an automation point of view, but can be a pain for the analyst. So this is a compromise to allow individual alerts to be aggregated into an anlyst friendly chunks.

Attributes

message_id

the smtp message id of the email that generated the alertgroup, if it exists This helps us in the reprocessing of mail inboxes

status

the status of the alertgroup: 'open', 'closed', 'promoted'

open_count

Number of open alerts in this alertgroup

closed_count

Number of closed alerts in this alertgroup

promoted_count

Number of promoted alerts in this alertgroup

alert_count

Number of alerts in this alertgroup

columns

used by ui to display alerts

firstview

this field is to track the first time someone viewed this alertgroup view_history, tracks the last time someone viewed, so not as useful for calculating response time metrics.

API will detect that value is -1 and then update the firstview to the seconds epoch when it was first viewed.

ahrefs

Some email messages contain HREFs back to the detection system that may be helpful to analysts evaluating the alert. This array will contain the <a href="xxx">xxx</a> for each one detected.
