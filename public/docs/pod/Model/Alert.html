Attributes

alertgroup

the integer id of the alertgroup this alert belongs in

status

the status of the alert from Scot::Role::Status valid statuses defined by Alertstatus in Types.pm

parsed

was this alert parsed (true = 1) otherwise we will need the original html from Scot::Role::Parsed

data

the hash reference to the data extraced from the alert from Scot::Role::Data

data_with_fair

same as data, but with detected entities wraped in spans (or as we say flaired) This is tricky though, we really only want to calculate flair when needed because of the expense

columns

the columns parsed from the data not sure this is needed in scot like it was in vast but tests break without it
